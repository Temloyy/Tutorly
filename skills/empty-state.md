# Empty state rules

All data-driven pages must include empty states.

Empty states must include:

* short message
* explanation
* primary action button

Empty states must guide users toward the next meaningful action.

---

## Tutorly-specific empty states

Examples include:

* No tutors found
* No bookings yet
* No upcoming sessions
* No completed sessions
* No reviews yet
* No saved tutors
* No notifications yet
* No analytics data yet
* No availability slots set (for tutors)
* No subjects added (for tutors)

---

## Behavior rules

* Empty states must be visually centered within the container
* Messaging must be clear and actionable
* Avoid generic phrases like “Nothing here”
* Always suggest what the user should do next

---

## Example structure

**Title:**
No bookings yet

**Explanation:**
You haven’t booked any sessions. Start by finding a tutor that matches your needs.

**Primary action:**
Find Tutors
